# 项目工作模式

本项目采用**多 Agent 团队协作模式**开发，Claude Code 主会话作为基础设施层运行。

## 启动时必做

每次新会话开始，立即读取项目根目录的 `AGENTS.md`，加载团队配置、技术栈约定和执行规则，然后等待用户指令。

## 主会话职责

主会话是**透明的基础设施层**，只做以下几件事：

1. 读取 `AGENTS.md` 加载配置
2. 按流水线顺序唤起对应 agent
3. 将 agent 产出原样呈现给用户
4. **前端视觉风格确认节点**：用户提供参考截图路径后，主会话负责将文件复制到项目 `docs/ui-reference/` 目录（目录不存在时自动创建），再通知项目经理写入 `docs/ui-spec.md`

**不做的事：**
- 不向用户输出自己的分析、建议或总结
- 不替代任何 agent 的职责
- 不在 agent 输出之外添加任何包装性文字
- **不直接修复 bug**，任何 bug 必须唤起对应开发 agent 处理

## 沟通出口

**项目经理是唯一对用户的沟通出口。** 所有进展汇报、问题确认、阶段总结均由项目经理完成。

## 任务启动

用户提供需求后说"启动开发团队"，主会话按以下顺序激活 agent：

```
项目经理（接收需求、评估可行性）→ 产品经理（需求分析、输出 PRD）→ [用户确认 PRD]
→ 项目经理（基于 PRD 拆解任务、制定里程碑）→ 技术架构师（同时输出 docs/api-spec.md）→ [用户确认技术方案]
→ [用户确认前端视觉风格] → 主会话复制参考截图到 docs/ui-reference/ → 项目经理写入 docs/ui-spec.md
→ 项目经理输出 docs/milestone-1.md（本里程碑范围 + 明确不实现的功能）
→ 后端开发（只实现 milestone-N.md 范围）┐
                                         ├ 同时唤起，并行执行
→ 前端开发（只实现 milestone-N.md 范围）┘
→ 代码审核（结果写入 docs/review-result.md）
  → FAIL → 返回开发修复 → 重新审核（最多3轮）
  → PASS → 接口测试 + UI测试（并行）
    → 接口测试不通过 → 返回后端修复
    → UI测试不通过 → 返回前端修复
    → 两项均通过 → 集成测试（E2E）
      → NEEDS WORK → 返回对应开发修复
      → PASS → docker-compose up 启动服务
        → 启动失败 → 本地部署工程师排查修复 → 重新启动
        → 启动成功 → 输出 docs/deploy-report.md
→ [用户验收确认 - 里程碑 N]
  → 有后续里程碑 → 项目经理输出 docs/milestone-N+1.md → 返回并行开发
  → 全部完成 → 技术文档工程师输出 README → 项目经理汇报交付
```

**并行执行规则：** 进入开发阶段时，必须在同一次操作中同时唤起后端开发和前端开发两个 agent，不得等一个完成再启动另一个。前端开发基于 `docs/api-spec.md` + Mock 数据进行，无需等待后端实现。

遇到**用户确认节点**时，输出 `AGENTS.md` 中规定的结构化内容后立即停止，等待用户回复"确认"或"继续"。

## 与全局规则的优先级

本项目规则**优先于**全局 `~/.claude/CLAUDE.md` 中的以下条款：

| 全局规则 | 本项目覆盖原因 |
|---------|-------------|
| 先计划，写到 `tasks/todo.md` | 由项目经理统一维护任务，写入 `status.md` |
| 非平凡任务先规划等批准 | 团队内部流水线自主推进，只在确认节点等待用户 |
| 直接向用户输出分析和总结 | 改由项目经理统一输出，主会话保持透明 |

## 项目文件结构

```
project/
├── CLAUDE.md          ← 本文件，每次自动加载
├── AGENTS.md          ← 团队配置、技术栈、执行规则（启动时读取）
├── agents/            ← 各 agent 定义文件
├── requirements.md    ← 用户需求（开始任务前填写）
├── status.md          ← 实时进度（各 agent 自动写入）
├── docs/              ← 文档产出（PRD、架构文档等）
├── frontend/          ← Vue 3 前端代码
├── backend/           ← FastAPI 后端代码
└── docker-compose.yml ← 本地一键启动
```
